# 📥 视频下载器 (HLS/m3u8) – 清晰度选择版(VideoDownloader_ClaritySelection_withCookie_m3u8_AES)

一个基于 Python 的图形化视频下载工具，支持通过浏览器自动提取网页中的 **m3u8/HLS** 视频流地址，并提供清晰度选择、Cookie 管理以及加密视频（AES-128-CBC）的解密下载功能。

## ✨ 功能特点

- **图形界面**：使用 Tkinter 构建，简单易用。
- **自动提取视频链接**：通过 Selenium 驱动 Chrome 浏览器，自动解析网页中的隐藏 m3u8 地址。
- **清晰度选择**：支持最高/最低画质或指定分辨率（1080p、720p 等），自动从多码率播放列表中挑选最合适的流。
- **Cookie 管理**：
  - 支持从本地保存的 Cookie 文件自动登录（首次需手动登录一次）。
  - 支持手动粘贴 Cookie 字符串，方便切换账号。
- **加密视频支持**：可解密 AES-128-CBC 加密的 TS 分片。
- **断点续传？**（目前不支持，但分片下载有重试机制，稳定性较高）
- **下载进度显示**：实时显示下载进度条和状态信息。

## 🖥️ 运行环境与依赖

### 1. Python 版本
- Python 3.7 及以上（建议 3.9+）

### 2. 浏览器与驱动
- **Google Chrome** 浏览器（需与系统匹配的最新稳定版）
- **ChromeDriver**：版本必须与你的 Chrome 版本一致。
  - 下载地址：https://chromedriver.chromium.org/
  - 下载后将 `chromedriver` 所在目录添加到系统 `PATH` 环境变量，或直接放在脚本同目录下。

### 3. Python 依赖库
使用以下命令安装所需库：

```bash
pip install selenium requests pycryptodome
```

> 📌 `tkinter` 是 Python 标准库，一般无需额外安装。Linux 系统若缺失可通过 `sudo apt install python3-tk` 安装。

## 🚀 使用方法

### 1. 启动程序
保存脚本为 `video_downloader.py`，然后在终端运行：
```bash
python video_downloader.py
```

### 2. 界面操作说明

| 组件 | 说明 |
|------|------|
| **视频页面地址** | 输入包含视频的网页 URL（例如某视频网站的播放页）。 |
| **保存到** | 选择下载文件的保存目录。默认为脚本所在目录下的 `downloads` 文件夹。 |
| **清晰度** | 下拉选择期望的分辨率（最高画质、1080p、720p…最低画质）。若视频源无匹配清晰度，将自动选择最接近的较低清晰度。 |
| **Cookie 来源** | 选择“使用本地保存的 Cookie”或“手动粘贴 Cookie 字符串”。 |
| **Cookie 字符串** | 仅当选择“手动粘贴”时需要填写。格式如 `key1=value1; key2=value2`。 |
| **重置本地 Cookie** | 删除已保存的本地 Cookie 文件，下次需重新登录。 |
| **开始下载** | 点击后自动执行：打开浏览器 → 处理登录 → 提取视频链接 → 下载并合并 TS 片段。 |

### 3. 首次使用（手动登录）
1. 在 **Cookie 来源** 中选择“使用本地保存的 Cookie”。
2. 点击 **开始下载**，会弹出一个 Chrome 窗口并导航到目标页面。
3. 若页面需要登录，请在浏览器中手动完成登录。
4. 登录成功后，回到程序弹出的消息框点击 **确定**。
5. 程序会自动保存 Cookie 并继续提取视频。

> 之后再次使用同一网站时，程序会自动加载本地 Cookie，无需重复登录。

### 4. 使用手动粘贴的 Cookie
1. 选择“手动粘贴 Cookie 字符串”。
2. 在浏览器中登录目标网站，使用开发者工具（F12）→ Application/存储 → Cookies，复制所有 Cookie 并整理为 `name=value; name2=value2` 格式。
3. 粘贴到“Cookie 字符串”文本框中，点击下载即可。

### 5. 下载过程
- 程序会先解析 m3u8 播放列表，若为多码率（master playlist）则根据选择的清晰度自动选取子流。
- 若视频流使用 AES-128 加密，程序会自动下载密钥并进行解密。
- 下载过程中会显示当前进度，完成后弹出成功提示。

## 📂 输出文件
- 最终合并的视频文件保存在你指定的目录下，文件名为 `video_output.mp4`。
- 临时分片文件会在下载完成后自动清理，不留痕迹。

## ⚠️ 注意事项

1. **法律合规**：请仅下载你有权获取的内容，遵守相关网站的服务条款及当地法律法规。
2. **链接时效性**：部分视频的 m3u8 地址可能很快失效（如直播回放），建议尽快下载。
3. **Chrome 驱动版本**：务必保证 Chrome 浏览器与 ChromeDriver 版本匹配，否则程序无法启动。
4. **反爬策略**：如果网站检测到自动化工具可能会禁止访问，可尝试注释代码中的 `headless` 模式或调整请求间隔。
5. **网络稳定性**：下载大视频时若网络中断，需重新下载（无断点续传）。程序内针对单个分片有最多 3 次重试，能应对短暂波动。

## 🧰 常见问题

### Q: 启动时提示 `selenium.common.exceptions.WebDriverException: Message: 'chromedriver' executable needs to be in PATH`
**A:** ChromeDriver 未配置。请下载对应版本的 `chromedriver` 并将其路径加入系统环境变量，或放到脚本同级目录。

### Q: 点击下载后浏览器打开了但没反应，或者提取不到视频链接
**A:** 可能是页面结构变化导致提取规则失效。尝试：
- 确认浏览器中确实可以正常播放视频。
- 检查 Cookie 是否有效（手动粘贴方式更可控）。
- 如果页面使用了更复杂的动态加载，可适当增加等待时间或修改 `_wait_for_video_ready` 中的逻辑。

### Q: 下载的视频没有声音或只有部分？
**A:** 部分网站将音频与视频分离存储。本工具目前只下载单一的 m3u8 流，若遇到音画分离的情况可能需要手动合并。

### Q: 想下载其他分辨率，但选项中没有怎么办？
**A:** 当前提供了常用分辨率选项，若视频源只有特定分辨率（如 1080p、360p 等），选择“最高画质”或“最低画质”即可覆盖所有可能。
